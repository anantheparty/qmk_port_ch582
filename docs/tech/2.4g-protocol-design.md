# 2.4G 私有协议设计

## 1. 概述

本文档描述 Obey65 键盘与 2.4G 接收器之间的私有无线协议设计。

## 2. 设计目标

- **低延迟**: < 5ms 端到端延迟
- **高可靠性**: 丢包率 < 0.1%
- **简单高效**: 协议开销小
- **安全性**: 基本的配对安全（非加密）

## 3. RF 配置

### 3.1 频段选择

```c
// 使用 CH582 的 2.4GHz RF
#define ESB_BASE_FREQ       2400    // 基础频率 2400MHz
#define ESB_CHANNEL_COUNT   16      // 可用信道数
#define ESB_DATA_RATE       2       // 2Mbps

// 频率跳变表 (避开 WiFi 密集区)
static const uint8_t esb_freq_table[ESB_CHANNEL_COUNT] = {
    2, 26, 50, 74,   // 避开 WiFi Ch 1, 6, 11
    5, 29, 53, 77,
    8, 32, 56, 80,
    11, 35, 59, 83
};
```

### 3.2 发射功率

```c
#define ESB_TX_POWER        LL_TX_POWEER_4_DBM  // 4dBm
```

## 4. 数据帧格式

### 4.1 通用帧结构

```
┌─────────────────────────────────────────────────────────────────┐
│  Preamble  │  Address  │  Length  │  Payload  │  CRC  │
│   (1 byte) │ (4 bytes) │ (1 byte) │ (N bytes) │(2 byte│
└─────────────────────────────────────────────────────────────────┘
```

| 字段 | 长度 | 说明 |
|------|------|------|
| Preamble | 1 byte | 前导码 0xAA |
| Address | 4 bytes | 设备地址 |
| Length | 1 byte | Payload 长度 |
| Payload | 1-32 bytes | 数据内容 |
| CRC | 2 bytes | CRC-16 校验 |

### 4.2 Payload 格式

```
┌──────────────────────────────────────────────┐
│  Type  │  Seq  │  Data  │
│ (1 byte)│(1 byte)│(0-30 bytes)│
└──────────────────────────────────────────────┘
```

### 4.3 消息类型定义

```c
typedef enum {
    ESB_MSG_PAIRING_REQ     = 0x01,  // 配对请求
    ESB_MSG_PAIRING_RSP     = 0x02,  // 配对响应
    ESB_MSG_PAIRING_ACK     = 0x03,  // 配对确认

    ESB_MSG_KEYBOARD        = 0x10,  // 键盘报告
    ESB_MSG_MOUSE           = 0x11,  // 鼠标报告
    ESB_MSG_CONSUMER        = 0x12,  // 媒体键报告

    ESB_MSG_ACK             = 0x80,  // 通用ACK
    ESB_MSG_NACK            = 0x81,  // 否定确认

    ESB_MSG_PING            = 0xF0,  // 心跳
    ESB_MSG_PONG            = 0xF1,  // 心跳响应
} esb_msg_type_t;
```

## 5. 协议交互

### 5.1 配对流程

```
   键盘                                      接收器
    │                                          │
    │  [用户按配对键]                          │ [用户按配对键]
    │                                          │
    │─────── PAIRING_REQ {kb_addr} ──────────►│
    │                                          │
    │◄────── PAIRING_RSP {rx_addr} ───────────│
    │                                          │
    │─────── PAIRING_ACK ────────────────────►│
    │                                          │
    │          [配对完成, 保存地址]             │
    │                                          │
```

配对请求 Payload:
```c
struct pairing_req {
    uint8_t type;           // ESB_MSG_PAIRING_REQ
    uint8_t kb_addr[4];     // 键盘地址
    uint16_t product_id;    // 产品ID
    uint8_t firmware_ver;   // 固件版本
};
```

### 5.2 键盘报告传输

```
   键盘                                      接收器
    │                                          │
    │─────── KEYBOARD {seq, report} ─────────►│
    │                                          │
    │◄────── ACK {seq} ───────────────────────│ (如果收到)
    │                                          │
    │   [如果 10ms 内未收到ACK, 重传]          │
    │                                          │
```

键盘报告 Payload:
```c
struct keyboard_report_msg {
    uint8_t type;           // ESB_MSG_KEYBOARD
    uint8_t seq;            // 序列号 (0-255 循环)
    uint8_t modifier;       // 修饰键
    uint8_t reserved;       // 保留
    uint8_t keys[6];        // 按键码
};
```

### 5.3 心跳机制

```
   键盘                                      接收器
    │                                          │
    │─────── PING ───────────────────────────►│ 每 500ms
    │                                          │
    │◄────── PONG ────────────────────────────│
    │                                          │
    │  [如果3次未收到PONG, 认为连接断开]       │
    │                                          │
```

## 6. 重传机制

### 6.1 重传策略

```c
#define ESB_MAX_RETRIES     3       // 最大重传次数
#define ESB_RETRY_DELAY_US  500     // 重传间隔 (微秒)
#define ESB_ACK_TIMEOUT_MS  10      // ACK 超时 (毫秒)

typedef struct {
    uint8_t seq;
    uint8_t retries;
    uint32_t send_time;
    uint8_t payload[32];
    uint8_t payload_len;
} esb_tx_buffer_t;
```

### 6.2 状态机

```
                    ┌─────────────────┐
                    │      IDLE       │
                    └────────┬────────┘
                             │ send()
                             ▼
                    ┌─────────────────┐
                    │   TX_PENDING    │◄───┐
                    └────────┬────────┘    │
                             │             │ timeout
                      ┌──────┴──────┐      │ && retries < MAX
                      │             │      │
                  ACK │             │ timeout
                      │             │      │
                      ▼             ▼──────┘
            ┌─────────────┐  ┌─────────────┐
            │  TX_SUCCESS │  │  TX_FAILED  │
            └─────────────┘  └─────────────┘
```

## 7. 地址管理

### 7.1 地址格式

```c
// 键盘地址: 基于芯片唯一ID生成
// 格式: {0xE7, chip_id[0], chip_id[1], chip_id[2]}
uint8_t keyboard_addr[4];

// 接收器地址: 出厂预设或用户生成
// 格式: {0xE7, rand[0], rand[1], rand[2]}
uint8_t receiver_addr[4];

// 广播地址 (配对时使用)
const uint8_t broadcast_addr[4] = {0xE7, 0xFF, 0xFF, 0xFF};
```

### 7.2 存储

```c
// EEPROM 存储配对信息
#define ESB_EEPROM_ADDR     0x100

typedef struct {
    uint32_t magic;         // 0xE5B12340
    uint8_t kb_addr[4];     // 键盘地址
    uint8_t rx_addr[4];     // 接收器地址
    uint8_t channel;        // 当前信道
    uint8_t paired;         // 是否已配对
    uint16_t crc;           // 校验
} esb_config_t;
```

## 8. 频率跳变

### 8.1 跳频策略

```c
// 简单的伪随机跳频
static uint8_t current_channel = 0;

void esb_hop_channel(void) {
    // 基于序列号的简单跳频
    current_channel = (current_channel + 7) % ESB_CHANNEL_COUNT;
    esb_set_frequency(esb_freq_table[current_channel]);
}

// 每次成功传输后跳频
void on_tx_success(void) {
    esb_hop_channel();
}
```

### 8.2 同步机制

键盘和接收器通过 ACK 帧中的信道信息保持同步。
